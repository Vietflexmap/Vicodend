# Security Policy

## Supported versions

Only the latest tagged release receives security updates.

## Reporting a vulnerability

Do not disclose exploitable vulnerabilities in a public issue. Use GitHub's private vulnerability reporting feature for this repository. Include the affected version, reproduction steps, impact and any proposed mitigation. Never include real passwords, access tokens, payment data or personal information in a report.

No software can guarantee absolute security. Operators are responsible for threat modeling, access control, monitoring, backups, updates and independent security assessment for their deployment.
